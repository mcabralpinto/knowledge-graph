*Two-person group project for the [[Knowledge & Language]] class in which we aim to make a chatbot that can answer questions about the Terraria wiki. Created in [[mcabralpinto (GH)]]*

## Initial Project Proposal
### Description
The aim of this project is to develop a conversational agent specialized in [[Terraria]]. Though conceptually simple, the game features complex mechanics that can be challenging for new players. Despite this, Terraria remains extremely popular, making it an ideal candidate for a [[LLM]] based chatbot to assist players of all skill levels. 
From a course perspective, this project explores how structured domain-specific knowledge (e.g., item statistics, progression systems) can be represented and extracted through natural language. The chatbot is therefore an applied case study for [[Knowledge Representation]] and language understanding on a restricted topic.
### System Overview
The architecture combines symbolic and statistical [[Natural Language Processing]]. User queries are analyzed to detect entities and intents, guiding semantic retrieval from a structured knowledge base. Responses are generated by the [[Ollama]] inference engine, merging factual data with natural phrasing.
**Pipeline:** user input → NLP parsing → [[RDF]] query → answer generation → optional confidence scoring.
### Knowledge Representation
Terraria entities (items, NPCs, biomes) will be modeled as nodes, with relations such as `drops`, `craftedFrom`, or `spawnsIn`.
Ontologies will be created in [[Protégé]], stored using [[RDFLib]], and queried through [[SPARQL]], allowing semantic lookups like “Which pre-Hardmode bosses drop weapons?”.
### Tools and Frameworks
- **Ollama -** Local LLM backend for fluent responses.
- **spaCy -** Entity and intent extraction.
- **RDFLib + SPARQL -** Knowledge storage and querying.
-  **Protégé -** Ontology design tool.
- **[[scikit-learn]] -** Confidence and uncertainty analysis.
### Steps
1. **Data Collection -** Provided by Terraria's wikis, and using [[BeautifulSoup]] to scrape relevant relevant information from pages.\\
2. **Data Transformation -** Wiki data will be turned into triplets to support efficient LLM access.
3. **Deployment (Optional) -** Integrate the chatbot into the in-game Guide NPC for a “Chatty Guide” experience.
