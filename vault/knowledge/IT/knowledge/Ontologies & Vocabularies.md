*Knowledge bank that represents information by defining concepts, properties and relationships.*

These two terms are often used interchangeably, but there is a subtle distinction:
- An ontology is a formal conceptualization of a domain
- A vocabulary is simply a set of terms and their meanings
In practice, however, both ontologies and vocabularies serve the same purpose of structuring knowledge (e. g. in a [[Knowledge Graph]]) in a way that can be understood and utilized by machines.
### Goals
- Represent a knowledge domain with semantics that machines can understand
- Share knowledge (between humans and machines)
- Reuse knowledge
- Make domain assumptions explicit, enable reasoning 
- Analyze domain knowledge 
- Enable large-scale machine processing
### Tools
- Can be built on top of [[RDF]] data
- Can be created in [[OWL]] format, using tools like [[Protégé]]
- Can be queried using [[SPARQL]]
