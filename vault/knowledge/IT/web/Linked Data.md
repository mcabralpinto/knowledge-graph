*Data published on the Web in such a way (structured, connected) that it is machine-readable. Application of the [[Semantic Web]].*
## The Basic Rules of Linked Data
Tim Berners-Lee outlined four principles for publishing Linked Data:
- **Use URIs as names for things:** Each entity (concept, object, etc.) should have a unique URI.
- **Use HTTP URIs:** This allows people and machines to look up those names on the web. When URIs identify real objects, they should be disambiguated from the web pages about them.
- **Provide useful information:** When someone looks up a URI, provide useful information using standards like [[RDF]] and [[SPARQL]].
- **Include links to other URIs:** This helps users discover more related information.
## FAIR Principles
Linked Data often adheres to the FAIR principles, which stand for:
- **Findable:** Data should be easy to find for both humans and computers. This includes using unique identifiers and rich metadata.
- **Accessible:** Once found, data should be retrievable using standard protocols.
- **Interoperable:** Data should be in a format that can be integrated with other data and work with various applications.
- **Reusable:** Data should be well-described so that it can be used in different contexts and for various purposes.
## Variants
- **Linked Open Data (LOD):** When Linked Data is made available under an open license, allowing anyone to use, modify, and share it freely.
