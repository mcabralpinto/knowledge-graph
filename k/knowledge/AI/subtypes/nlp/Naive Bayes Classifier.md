*[[Machine Learning]] classification algorithm that applies [[Bayes’ Theorem#Naive Bayes|Naive Bayes' theorem]].*
### [[Text Classification]]
Uses [[Bayes’ Theorem]] to classify text based on the probabilities of words occurring in different sentiment classes (e.g., positive, negative, neutral). It calculates the posterior probability for each class given the input text, assuming independence between features (words) and that position in the text does not matter (bag-of-words). Given a document $d$ and class $c$: $$ P(c|d) = \frac{P(d|c) \times P(c)}{P(d)} $$
To classify each word, we want to find $c$ with maximum $P(c|d)$. This can be simplified to: $$ c = argmax_{c ∈ C} P(c) \prod_{i=1}^{n} P(w_i|c) $$But, given that $w_i$ are the words in document $d$, what is $P(w_i|c)$? It can be estimated from the training data as: $$ P(w_i|c) = \frac{count(w_i, c) + 1}{\sum_{w ∈ V} (count(w, c) + 1)} $$where $count(w_i, c)$ is the number of times word $w_i$ appears in documents of class $c$, and $V$ is the vocabulary (all unique words in the training set). The "+1" is for Laplace smoothing to avoid zero probabilities. But this assumes all words are equally important - a $weight$ function can be used instead of $count$ to give different importance to words using **[[Text Classification#**TF-IDF (Term Frequency-Inverse Document Frequency)**|TF-IDF]]**.
**For Knowledge & Language exercises**, extract vocabulary $V$ from the training corpus, compute $P(c) = \frac{|c_{j}~docs|}{N}$ and $P(w_i|c) = \frac{n_{k} + \alpha}{n + \alpha N}$ for each class $c$ and word $w_i$ ($n_i$ is how many times $w_i$ appears in $c_j$ docs, $n$ how many unique words there are in $c_j$ docs), ignoring unknown words and stopwords, then classify new documents using the $c = \dots$ formula above.
